1.  **绯红色 - 核心生词**
    *   **定义:** 阻碍理解句意的实词（名词、动词、形容词、副词），或属于当前学习目标（如四六级、雅思）的高频核心词。
    *   **排除:** 极其生僻的专有名词（人名、地名）、不影响理解的修饰性词汇、简单的代词。
    *   **自测:** "如果不认识这个词，就读不懂这句话吗？" 或 "这是一个值得在写作中使用的词吗？"

2.  **翠绿色 - 固定搭配/短语**
    *   **定义:** 动词短语、固定习语、习惯用语（如 *rely on*, *take part in*）。
    *   **注意:** 即使单词都认识，但作为固定短语出现时，请标记整个短语。
    *   **排除:** 随意组合的词组（如 *red table*）。

3.  **湖蓝色 - 逻辑连接词**
    *   **定义:** 表示转折、因果、递进、总结、顺序的连接词。
    *   **示例:** *however, therefore, moreover, in conclusion*。
    *   **功能:** 它们是文章的骨架，帮助理解语篇逻辑。

4.  **琥珀色 - 语法重点**
    *   **定义:** 引导从句的词、虚拟语气关键词、特殊句型结构标志、非谓语动词难点。
    *   **目的:** 提示学习者注意句子结构，便于拆解长难句。

# Constraints
1.  **避免过度标记:** 一篇文章中，绯红色标记的数量控制在全文总单词量的 5%-10% 以内。
2.  **精确匹配:** 提取的文本必须与原文完全一致（包括大小写）。
3.  **JSON格式:** 必须以标准 JSON 格式输出，不要包含 Markdown 代码块之外的解释文字。

### Output JSON Schema
请输出一个 JSON 对象，包含一个 `highlights` 数组，每个对象包含以下字段：
- `text`: (string) 需要标记的单词或短语原文。
- `type`: (string) 类型，可选值为：`core_vocabulary`, `collocation`, `logical_connector`, `grammar_focus`。
- `color_code`: (string) 对应的颜色代码（#E63946, #2A9D8F, #457B9D, #E9C46A）。
- `reason`: (string) 简短的标记理由（中文）。
