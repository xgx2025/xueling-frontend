# Skill: elegant-english-reading-experience

## Description
Design a high-fidelity, comfortable, and aesthetically pleasing reading interface for English articles. This style prioritizes typography, whitespace, and information hierarchy over decorative elements. It avoids generic "AI-generated" aesthetics (e.g., excessive gradients, standard tech blues, or heavy border-radius) in favor of a clean, editorial, or "human-crafted" feel.

## Goals
1.  **Readability First**: Optimize for eye comfort and scanning speed for English text.
2.  **Editorial Aesthetic**: Achieve a high-quality look similar to Medium, The New Yorker, or technical documentation like Stripe.
3.  **Clear Hierarchy**: Make metadata, headings, body, and callouts distinct at a glance.
4.  **Avoid Clichés**: No default blue links, no heavy shadows, no excessive generic rounded corners.

## Design Tokens & Principles

### 1. Color Palette (Warm & Paper-like)
Avoid pure white (`#FFFFFF`) backgrounds and stark blacks. Use a subtle "warm paper" base to reduce eye strain.

*   **Background (Surface)**: `#F9F8F4` (Off-white / Eggshell) or `#FAFAF9` (Stone-50).
*   **Text (Primary)**: `#292524` (Stone-800) or `#1C1917` (Stone-900). Never pure `#000`.
*   **Text (Secondary/Metadata)**: `#78716C` (Stone-500).
*   **Accent (Links/Highlights)**: `#EA580C` (Burnt Orange) or `#059669` (Emerald). *Avoid default `#0000EE` blue.*
*   **Borders/Separators**: `#E7E5E4` (Stone-200). Use thin lines (`1px solid`).

### 2. Typography System (Vertical Rhythm)
English text needs specific line-height and line-length to be readable.

*   **Font Family**:
    *   *Headings*: A serif font (e.g., *Playfair Display*, *Merriweather*, or system-ui-serif) for elegance.
    *   *Body*: A geometric humanist sans-serif (e.g., *Inter*, *SF Pro Text*, *Optima*) or a high-legibility serif (e.g., *Lora*, *Georgia*). Let's default to **Serif for Body** (classic editorial) or **Sans-serif with wide tracking** (modern documentation).
*   **Sizing (Scale)**:
    *   `--font-sm`: 0.875rem (Metadata, captions)
    *   `--font-base`: 1.125rem (Body text - slightly larger than default 1rem)
    *   `--font-lg`: 1.5rem (H3)
    *   `--font-xl`: 2.25rem (H2)
    *   `--font-2xl`: 3rem (H1)
*   **Line Height**:
    *   Body: `1.7` to `1.8` (Crucial for English reading flow).
    *   Headings: `1.2` to `1.3`.
*   **Line Length (Measure)**:
    *   Max-width for body paragraphs: `65ch` to `75ch` (approx 600px - 700px). Never let text span the full screen width.
*   **Letter Spacing**:
    *   Uppercase labels (metadata): `0.05em` to `0.1em`.

### 3. Layout & Spacing (The "Breath" Design)
Avoid clutter. Use whitespace as a structural element.

*   **Container**: Centered with `max-width: 720px` for the main reading column.
*   **Vertical Rhythm**: Use a base unit of `8px` or `0.5rem`.
    *   Margin-bottom for paragraphs: `1.5rem`.
    *   Margin-bottom for headings: `1rem`.
*   **Metadata**: Place above the title (Author, Date, Read Time) in a smaller, uppercase, tracked-out style. Use a subtle divider line.

### 4. Component Details (Anti-AI Clichés)

#### Headings (H1-H3)
*   **Style**: Tighter line height than body. Use a Serif font for H1/H2 to create a "anchor" for the eye.
*   **Weight**: 600 or 700. Avoid 400 (too weak) or 900 (too heavy).

#### Paragraphs
*   **Indentation**: None. Use margin-bottom for separation.
*   **First Paragraph**: Can optionally be slightly larger (`drop-cap` effect or just larger font size).

#### Links
*   **Style**: Do not use standard underline.
*   **State**:
    *   Default: Accent color, no underline.
    *   Hover: Add a background underline (`text-decoration: underline; text-decoration-thickness: 2px; text-underline-offset: 3px; text-decoration-color: [accent-color-with-opacity];`).

#### Blockquotes
*   **Style**: No heavy background colors.
*   **Border**: A `3px solid` left border in the Accent color.
*   **Font**: Italic, maybe slightly larger size (`1.25rem`).
*   **Icon**: Optional large serif quote mark in background with low opacity.

#### Code Blocks
*   **Style**: Avoid default dark grey boxes.
*   **Background**: `#F3F4F6` (Cool Gray 100) or a very desaturated version of the accent color.
*   **Border**: `1px solid #E5E7EB`.
*   **Radius**: `4px` (Small, sharp radius). Do not use `12px` or `16px` (looks too "app-like").
*   **Font**: Monospace (`'Fira Code'`, `'JetBrains Mono'`).

#### Images
*   **Style**: Full width of the container, rounded `2px` or `0px` (sharp edges look more editorial).
*   **Caption**: Centered, `--font-sm`, `--color-secondary`, placed `0.5rem` below the image.

## Implementation Rules (CSS Guide)
When generating code, follow these constraints:

1.  **Use CSS Variables**: Define colors and spacing at `:root`.
2.  **Fluid Typography**: Use `clamp()` for font sizes to ensure responsiveness.
3.  **Proper Semantic HTML**: Use `<article>`, `<header>`, `<section>`, `<aside>` (for TOC).
4.  **TOC (Table of Contents)**: If present, place it on the right side (desktop) or top (mobile). Sticky positioning is good.

## Examples of "Bad" vs "Good"

| Element | ❌ Bad (Generic AI Style) | ✅ Good (This Skill) |
| :--- | :--- | :--- |
| **Background** | Pure White `#FFF` | Warm Paper `#F9F8F4` |
| **Link Color** | Bright Blue `#0000FF` | Burnt Orange `#EA580C` |
| **Buttons** | Gradient, heavy shadow, `rounded-full` | Solid outline, sharp corners, `hover:underline` |
| **Cards** | White bg on white bg, `shadow-xl`, `rounded-xl` | No card background, just text with minimal divider lines |
| **Font** | System Default Sans | Serif Headings + Optimized Sans Body |
